Algorithm Workflows & Flowcharts

Detailed workflows, PlantUML flowcharts, and pseudocode for all the optimization algorithms implemented in the Travel Route File-Based System.

1. Genetic Algorithm (GA)

Workflow

  • Initialize: Generate an initial population of random, valid routes.
  • Evaluate: Calculate fitness for all routes.
  • Evolve: Loop for N generations:
    • Elitism: Carry over the top k routes to the next generation without changes.
    • Selection: Use tournament selection to pick parents.
    • Crossover: Combine parts of two parents to create a child route (Order Crossover).
    • Mutation: Randomly swap, reverse, or replace places/hotels in the child route.
    • Replace: The new population replaces the old one.
  • Result: Return the best route found across all generations.

Flowchart

@startuml
start
:Initialize Population N=100;
:Evaluate Fitness;
while (Gen < 200?) is (Yes)
  :Copy Top 5 Elite Routes;
  while (New Pop < 100?) is (Yes)
    :Tournament Selection x2;
    :Order Crossover prob=0.8;
    :Mutation prob=0.3;
    :Evaluate Child & Add to New Pop;
  endwhile (No)
  :Update Global Best Route;
endwhile (No)
:End: Return Best;
stop
@enduml

Pseudocode

// 1. Initialization Create initial population P = GenerateRandomPopulation(size=100) Evaluate fitness for each route in P best_route = MinFitness(P) Generation = 1 // 2. Evolution Loop WHILE Generation <= 200 DO: Create new_population P_new // a. Elitism Add Top 5 Elite from P to P_new WHILE size(P_new) < 100 DO: // b. Selection Select parents P1, P2 from P based on fitness (Tournament, size=5) // c. Crossover Apply Order Crossover to P1, P2 to create offspring child (rate=0.8) // d. Mutation Apply mutation operator to child (rate=0.3) // e. Evaluation & Replacement Evaluate fitness for child and add to P_new END WHILE // f. Update Population & Best Solution Replace old population P with P_new Update best_route = MinFitness(P ∪ {best_route}) // g. Update Generation Counter Generation = Generation + 1 END WHILE // 3. Return Best Solution RETURN best_route

2. Simulated Annealing (SA)

Workflow

  • Initialize: Generate a random initial route and set a starting temperature.
  • Iterate per Temp: For a fixed number of iterations, explore the neighborhood.
  • Neighborhood Move: Swap, reverse, or replace places/hotels to generate a neighbor.
  • Acceptance: If the neighbor is better, accept it. If worse, accept it probabilistically based on the temperature.
  • Cooling: Multiply the temperature by a cooling rate.
  • Result: Return the best route found when the minimum temperature is reached.

Flowchart

@startuml
start
:Init Route, Temp=100;
while (Temp > 0.01?) is (Yes)
  while (Iter < 50?) is (Yes)
    :Generate Random Neighbor;
    :Delta = Neighbor - Current;
    if (Delta < 0?) then (Yes)
      :Accept & Update Current;
      :Update Global Best;
    else (No)
      if (Rand() < exp(-Delta/Temp)?) then (Yes)
        :Accept & Update Current;
        :Update Global Best;
      else (No)
      endif
    endif
  endwhile (No)
  :Temp = Temp * 0.995;
endwhile (No)
:End: Return Best;
stop
@enduml

Pseudocode

// 1. Initialization Create initial current_route = GenerateRandomRoute() best_route = current_route Set initial temperature Temp = 100.0 // 2. Annealing Loop WHILE Temp > 0.01 DO: // a. Iterations at current temperature FOR Iteration from 1 to 50 DO: // b. Neighborhood Move Generate neighbor from current_route (randomly swap/reverse/replace) // c. Evaluation Calculate delta = Fitness(neighbor) - Fitness(current_route) // d. Acceptance Criterion IF delta < 0 OR Random() < exp(-delta / Temp) THEN current_route = neighbor // e. Update Best Solution IF Fitness(current_route) < Fitness(best_route) THEN best_route = current_route END IF END IF END FOR // f. Cooling Schedule Temp = Temp * 0.995 END WHILE // 3. Return Best Solution RETURN best_route

3. Saving Method (SM - Clarke-Wright Heuristic)

Workflow

  • Select Hotels: Greedily select the best hotel(s) based on average distance to the depot and tourist spots.
  • Compute Savings: For each day, compute the distance saved by combining two places rather than visiting them separately from the hub.
  • Construct Route: Sort the savings and greedily append pairs of places into the day's route, respecting constraints.
  • Optimize Order: Use a nearest-neighbor approach to finalize the ordering of places within each day.

Flowchart

@startuml
start
:Select Optimal Hotels;
while (For Each Day?) is (Yes)
  :Compute Clarke-Wright Savings;
  :Sort Savings Descending;
  :Add 1 OTOP & 1 Food Place;
  :Greedily Add Places via Savings;
  if (Valid Limits & Time?) then (Yes)
    :Add Place to Day Route;
  else (No)
    :Check Next Saving Pair;
  endif
  :Fill Remaining Slots by Rate;
  :Nearest-Neighbor Ordering;
  :Store Day Route;
endwhile (No)
:Combine Days & Hotels;
:End: Return Final Route;
stop
@enduml

Pseudocode

// 1. Initialization Select optimal hotel_ids based on average distance (trip_days - 1) Initialize empty list route_days // 2. Day-by-Day Construction Loop FOR day = 1 to trip_days DO: Set hub = Start location for day Set end = End location for day Define available_places = All candidates excluding already used // a. Savings Calculation Compute Clarke-Wright savings for all pairs from hub Sort savings in descending order // b. Initial Day Route Initialize day_route with best OTOP place for hub // c. Greedily Add Places FOR EACH pair IN sorted savings DO: IF pair fits within max_places AND time_window THEN Append pair to day_route END IF END FOR // d. Fill Remaining Slots IF length(day_route) < max_places THEN Append best rated unused places to day_route END IF // e. Finalize Order Apply Nearest-Neighbor ordering to day_route Append day_route to route_days END FOR // 3. Return Best Solution RETURN Route(route_days, hotel_ids)

4. Genetic Algorithm + ALNS (GA+ALNS)

Workflow

  • Base: Uses the standard GA framework.
  • Enhancement: Integrates ALNS (Adaptive Large Neighborhood Search) as a local search mechanism.
  • Elite Polish: The top k elite routes are passed through the ALNS optimizer for refinement every generation.
  • Child Polish: 30% of newly generated children are passed through the ALNS optimizer before being added to the population.

Flowchart

@startuml
start
:Initialize Pop N=50;
while (Gen < 80?) is (Yes)
  :Apply ALNS Local Search to Top 5 Elite;
  while (New Pop < 50?) is (Yes)
    :Tournament + Crossover;
    :Mutation;
    if (Rand < 0.3?) then (Yes)
      :Apply ALNS Local Search to Child;
    else (No)
    endif
    :Add to New Pop;
  endwhile (No)
  :Update Global Best;
endwhile (No)
:End: Return Best;
stop
@enduml

Pseudocode

// 1. Initialization Create initial population P = GenerateRandomPopulation(size=50) Evaluate fitness for each route in P best_route = MinFitness(P) Generation = 1 // 2. Evolution Loop with ALNS Enhancement WHILE Generation <= 80 DO: Create new_population P_new // a. Elite Polish (ALNS Local Search) FOR EACH elite IN Top 5 of P DO: Apply ALNS Local Search (10 iterations) to elite Add improved_elite to P_new END FOR WHILE size(P_new) < 50 DO: // b. Selection Select parents P1, P2 using Tournament Selection // c. Crossover & Mutation Apply Crossover to P1, P2 to create child Apply Mutation to child // d. Child Polish (ALNS Local Search) IF Random() < 0.3 THEN Apply ALNS Local Search (10 iterations) to child END IF // e. Evaluation & Replacement Evaluate fitness for child and add to P_new END WHILE // f. Update Population & Best Solution Replace old population P with P_new Update best_route = MinFitness(P ∪ {best_route}) // g. Update Generation Counter Generation = Generation + 1 END WHILE // 3. Return Best Solution RETURN best_route

5. Simulated Annealing + ALNS (SA+ALNS)

Workflow

  • Base: Follows the SA temperature and acceptance schema.
  • Neighborhood Generation: Instead of random moves, neighbors are generated by applying one Destroy operator and one Repair operator.
  • Adaptive Weights: Operators are selected based on roulette wheel selection (weights). Weights are updated based on the historical success of the operators (adaptive).

Flowchart

@startuml
start
:Init Route, Temp=50, Weights=1;
while (Temp > 0.1?) is (Yes)
  while (Iter < 15?) is (Yes)
    :Select Destroy & Repair Ops via Weights;
    :Apply Destroy -> Repair;
    :Delta = Neighbor - Current;
    if (Delta < 0 or Prob?) then (Yes)
      :Accept Neighbor & Score Ops;
    else (No)
      :Reject;
    endif
  endwhile (No)
  :Update Op Weights per 100 iters;
  :Temp = Temp * 0.99;
endwhile (No)
:End: Return Best;
stop
@enduml

Pseudocode

// 1. Initialization Create initial current_route = GenerateRandomRoute() best_route = current_route Set initial temperature Temp = 50.0 Initialize destroy_weights and repair_weights (all 1.0) Total_Iterations = 0 // 2. Annealing & ALNS Loop WHILE Temp > 0.1 DO: // a. Iterations at current temperature FOR Iteration from 1 to 15 DO: // b. Adaptive Operator Selection Select destroy_op based on destroy_weights Select repair_op based on repair_weights // c. Neighborhood Move (Destroy & Repair) Apply destroy_op then repair_op to current_route to create neighbor // d. Evaluation Calculate delta = Fitness(neighbor) - Fitness(current_route) // e. Acceptance & Scoring Set accept = False IF delta < 0 THEN accept = True Increase scores for destroy_op and repair_op (+3 or +2) ELSE IF Random() < exp(-delta / Temp) THEN accept = True Increase scores for destroy_op and repair_op (+1) END IF // f. Update Route IF accept is True THEN current_route = neighbor IF Fitness(current_route) < Fitness(best_route) THEN best_route = current_route END IF END IF Total_Iterations = Total_Iterations + 1 END FOR // g. Adaptive Weights Update IF Total_Iterations is multiple of 100 THEN Update destroy_weights and repair_weights based on scores END IF // h. Cooling Schedule Temp = Temp * 0.99 END WHILE // 3. Return Best Solution RETURN best_route

6. Saving Method + ALNS (SM+ALNS)

Workflow

  • Phase 1 (Construction): Run the deterministic SM optimizer to rapidly build a high-quality initial solution.
  • Phase 2 (Refinement): Pass the SM route into an ALNS loop.
  • ALNS Loop: For N iterations, randomly destroy and repair the route, greedily accepting any improvements.

Flowchart

@startuml
start
:Phase 1: Run SM Optimizer;
:Init ALNS Loop with SM Route;
while (Iter < 100?) is (Yes)
  :Select Random Destroy & Repair Ops;
  :Destroy -> Repair;
  if (Fitness < Current?) then (Yes)
    :Accept & Update Best;
  else (No)
    :Reject;
  endif
endwhile (No)
:End: Return Best;
stop
@enduml

Pseudocode

// 1. Initialization (Phase 1: Construction) Create initial current_route = Run SM_Optimize() to generate high-quality base best_route = current_route // 2. Refinement Loop (Phase 2: ALNS) FOR Iteration from 1 to 100 DO: // a. Operator Selection Select random destroy_op (Random, Worst, Shaw) Select random repair_op (Greedy, Random, Regret) // b. Neighborhood Move (Destroy & Repair) Apply destroy_op then repair_op to current_route to create neighbor // c. Evaluation & Acceptance IF Fitness(neighbor) < Fitness(current_route) THEN current_route = neighbor // d. Update Best Solution IF Fitness(current_route) < Fitness(best_route) THEN best_route = current_route END IF END IF END FOR // 3. Return Best Solution RETURN best_route

7. Multi-Objective Memetic Algorithm (MOMA)

Workflow

  • Initialization: Create an initial population where 10% are derived from the Saving Method (SM) for rapid convergence, and the rest are generated randomly.
  • NSGA-II Backbone: Evaluate all routes across three distinct objectives (Distance, CO2, and negative Rating). Do not collapse these into a single scalar fitness. Instead, assign a Non-Dominated Pareto Rank and a Crowding Distance to each route.
  • Strict Elitism: At the start of each generation, the top routes (from the 1st Pareto Front) are explicitly protected and carried over without any mutation.
  • Reproduction & ALNS: Generate offspring via Tournament Selection (preferring better rank and wider crowding distance) and Order Crossover. Apply ALNS (Memetic local search) exclusively to the offspring.

Flowchart

@startuml
start
:Phase 1: Initialization;
:Generate 10% of Pop using SM (Saving Method);
:Generate 90% of Pop Randomly;
:Evaluate all solutions (Distance, CO2, Rating);

while (Gen < Max Generations?) is (Yes)
  :Phase 2: NSGA-II Evaluation;
  :Non-dominated Sorting (Fast Pareto Ranking);
  :Calculate Crowding Distance;
  
  :Phase 3: Strict Elitism;
  :Copy Top N Elites (Rank 1) to New Pop;

  :Phase 4: Reproduction & Genetic Ops;
  while (New Pop < Pop Size?) is (Yes)
    :Tournament Selection (Rank & Crowding);
    :Order Crossover;
    if (Rand < ALNS_Mutation_Rate?) then (Yes)
      :Phase 5: Memetic Local Search (ALNS);
      :Select Destroy & Repair Ops;
      :Apply ALNS to Child;
    else (No)
      :Standard Random Mutation;
    endif
    :Add Child to New Pop;
  endwhile (No)
  
  :Merge Parent + Child Populations;
  :Keep top N using NSGA-II elitism;
endwhile (No)

:End: Return Best Scalar Route from 1st Pareto Front;
stop
@enduml

Pseudocode

// 1. Initialization (Phase 1: Hybrid Seed) Create empty population P P_SM = Generate routes using SM_Optimize() (10% of Pop size) P_Random = GenerateRandomPopulation(90% of Pop size) P = P_SM ∪ P_Random Evaluate multiple objectives for each route in P (F1: Dist, F2: CO2, F3: -Rating) Generation = 1 // 2. Evolution Loop (Phase 2: NSGA-II Backbone) WHILE Generation <= Max_Generations DO: Create new_population P_child // a. Strict Elitism Assign Pareto Ranks using Non-dominated Sorting on P Copy Top N Elites directly from P to P_child // b. Reproduction WHILE size(P_child) < Pop_Size DO: // Selection based on Pareto Rank and Crowding Distance Select parents P1, P2 using Binary Tournament Child = Order_Crossover(P1, P2) // c. Memetic Local Search (Phase 3: ALNS Injection) IF Random() < ALNS_Mutation_Rate THEN destroy_op = Select Random Destroy repair_op = Select Random Repair Child = repair_op(destroy_op(Child)) ELSE IF Random() < Standard_Mutation_Rate THEN Child = Mutate_Swap_or_Reverse(Child) END IF Add Child to P_child END WHILE // d. Environmental Selection P_combined = P ∪ P_child Assign Pareto Ranks on P_combined Calculate Crowding Distance P = SelectTopN(P_combined, Pop_Size) Generation = Generation + 1 END WHILE // 3. Return Best Solution RETURN best_balanced_route_from(Pareto_Front(P))

8. ALNS Operators Reference

Destroy Operators

  • Random Removal: Randomly selects N places and removes them from the route. Promotes broad exploration.
  • Worst Removal: Iteratively evaluates the cost reduction of removing each place, and removes the N places that improve the fitness the most.
  • Shaw Removal: Removes places that are geographically close to each other (based on distance matrix) to reorganize entire local clusters.

Repair Operators

  • Greedy Insert: Tests every removed place at every possible insertion point and iteratively inserts the place that results in the lowest immediate fitness cost.
  • Random Insert: Inserts removed places back into random valid positions across any day.
  • Regret Insert: Evaluates the cost of inserting a place at its best position vs its second-best position (the "regret"). Inserts the place with the highest regret first, delaying easier decisions.
